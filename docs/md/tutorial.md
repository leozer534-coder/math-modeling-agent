## 第一部分：快速入门

### 1.1 什么是 MathModelAgent？

MathModelAgent 是一个专为数学建模设计的 AI Agent 系统，能够：

- 🔍 自动分析问题，建立数学模型
- 💻 自动编写和运行代码求解
- 📝 自动生成完整的建模论文
- 🤖 多 Agent 协作（协调者、建模者、代码者、写作者）

### 1.2 5 分钟快速开始

**步骤 1: 启动服务**
```bash
git clone https://github.com/leozer534-coder/math-modeling-agent.git
cd math-modeling-agent
docker-compose up
```

**步骤 2: 访问界面**
- 前端：http://localhost:5173
- 后端：http://localhost:8000

**步骤 3: 配置 API Key**
- 点击左侧边栏头像
- 输入 DeepSeek API Key
- 保存

**步骤 4: 运行第一个示例**
```
问题：某农场有 50 亩土地，可种植玉米和小麦。
玉米每亩产量 800kg，成本 500 元，售价 2 元/kg；
小麦每亩产量 600kg，成本 400 元，售价 2.5 元/kg。
如何分配种植面积使利润最大？
```

## 第二部分：核心功能详解

### 2.1 多 Agent 协作系统

#### 协调者 (Coordinator)
- 角色：任务分解和调度
- 职责：
  - 理解用户问题
  - 分解为子任务
  - 分配给其他 Agent
  - 整合最终结果

#### 建模者 (Modeler)
- 角色：数学建模专家
- 职责：
  - 分析问题背景
  - 选择合适的数学模型
  - 建立数学公式
  - 设计求解方法

#### 代码者 (Coder)
- 角色：编程实现专家
- 职责：
  - 根据模型编写代码
  - 调试和修正错误
  - 优化代码性能
  - 生成可视化图表

#### 写作者 (Writer)
- 角色：论文撰写专家
- 职责：
  - 整理解题过程
  - 撰写规范论文
  - 格式化排版
  - 生成参考文献

### 2.2 工作流程

```
用户问题
   │
   ▼
┌─────────────┐
│  协调者分析  │
└──────┬──────┘
       │
   ┌───▼───┐
   │任务分解│
   └───┬───┘
       │
   ┌───▼──────────┐
   │建模者 → 模型  │
   └──────┬───────┘
          │
   ┌──────▼───────┐
   │代码者 → 求解  │
   └──────┬───────┘
          │
   ┌──────▼───────┐
   │写作者 → 论文  │
   └──────┬───────┘
          │
          ▼
      用户
```

## 第三部分：进阶使用

### 3.1 自定义 Prompt 模板

位置：`backend/app/config/md_template.toml`

**示例:**
```toml
[abstract]
background = "第一段：背景介绍，约 100 字"
method = "第二段：针对问题一，使用什么模型，得到什么结果"
conclusion = "最后一段：总结和敏感性分析"

[keywords]
count = 5
example = "HHO MOHHO 遗传算法 Mealpy 蒙特卡洛模拟"
```

### 3.2 配置不同 LLM 模型

**按 Agent 配置:**
```env
# 协调者 - 使用强推理模型
COORDINATOR_MODEL=claude-3-5-sonnet-20241022
COORDINATOR_API_KEY=sk-ant-xxx

# 建模者 - 使用数学强的模型
MODELER_MODEL=deepseek/deepseek-chat
MODELER_API_KEY=sk-ds-xxx

# 代码者 - 使用代码强的模型
CODER_MODEL=gpt-4o
CODER_API_KEY=sk-xxx

# 写作者 - 使用性价比模型
WRITER_MODEL=deepseek/deepseek-chat
WRITER_API_KEY=sk-ds-xxx
```

### 3.3 使用云端代码执行器

**配置 E2B:**
```env
E2B_API_KEY=your-e2b-api-key
```

优势：
- 更安全（沙箱环境）
- 更强大（可安装任意包）
- 更稳定（不受本地环境影响）

### 3.4 调整运行参数

```env
# 最大对话轮次（增加可提高质量但更慢）
MAX_CHAT_TURNS=100

# 最大重试次数
MAX_RETRIES=5

# 日志级别
LOG_LEVEL=DEBUG

# 调试模式
DEBUG=true
```

## 第四部分：最佳实践

### 4.1 问题描述技巧

**好的描述:**
```
问题：线性规划优化

某工厂生产 A、B 两种产品：
- 产品 A：需要原料 1 为 2kg，原料 2 为 1kg，工时 3 小时，利润 5 元/件
- 产品 B：需要原料 1 为 1kg，原料 2 为 2kg，工时 2 小时，利润 4 元/件

可用资源：
- 原料 1：100kg/天
- 原料 2：80kg/天
- 工时：120 小时/天

目标：最大化总利润

请建立线性规划模型并求解。
```

**不好的描述:**
```
帮我做个优化题
```

### 4.2 结果验证

AI 生成的结果需要人工验证：

1. **模型合理性**: 检查假设是否符合实际
2. **计算正确性**: 验证代码逻辑和结果
3. **论文完整性**: 检查摘要、关键词、参考文献

### 4.3 迭代优化

如果第一次结果不理想：

1. 补充更多背景信息
2. 明确指出需要改进的地方
3. 提供示例或参考格式
4. 调整模型参数

**示例:**
```
上一个回答中，模型建立得很好，但是：
1. 请增加敏感性分析
2. 补充更多的图表可视化
3. 参考文献格式改为 GB/T 7714
```

### 4.4 性能优化

**提高响应速度:**
- 使用国内 API 提供商（DeepSeek、通义千问）
- 减少 MAX_CHAT_TURNS
- 使用更快的模型

**提高生成质量:**
- 使用更强的模型（Claude 3.5、GPT-4o）
- 增加 MAX_CHAT_TURNS
- 提供更多背景信息

## 第五部分：常见问题

### Q1: 生成的论文可以直接提交比赛吗？

**A:** 目前 AI 生成的论文**不建议直接提交**，应该：
- 人工审核和修改
- 补充个人思考和创新
- 验证模型和结果的正确性
- 调整格式符合比赛要求

AI 是辅助工具，不是替代品。

### Q2: 如何处理复杂问题？

**A:** 复杂问题分解为多个子问题：
1. 先解决核心模型
2. 逐步添加约束条件
3. 分阶段求解
4. 最后整合结果

### Q3: 代码执行报错怎么办？

**A:** 
1. 查看错误日志
2. 让 AI 自动修正（通常 1-2 次即可）
3. 手动检查代码逻辑
4. 简化问题重新尝试

### Q4: 如何保存和导出结果？

**A:**
```bash
# 查看生成的文件
ls backend/project/work_dir/xxx/

# 复制论文
cp backend/project/work_dir/xxx/res.md ./my_paper.md

# 转换为 PDF
pandoc my_paper.md -o my_paper.pdf
```

### Q5: 支持哪些编程语言？

**A:** 目前主要支持：
- ✅ Python（主要支持）
- ⏳ R 语言（计划中）
- ⏳ MATLAB（计划中）
